## Routes

### Route Strategy

Use App Router with static rendering by default. All migrated URL paths from `url_inventory.json` status `200` remain canonical pages at the same path. This is the SEO-preserving route strategy.

Important conflict: the task text asks to confirm `/blog/[slug]` and `/services/[slug]`, but the baseline has high-value posts and service pages at root-level paths such as `/pt-141/`, `/hormone-therapy/`, `/vaginal-rejuvenation-expert/`, and `/5-tips-for-achieving-orgasm-through-intercourse/`. Moving these to `/blog/[slug]` or `/services/[slug]` would create hundreds of redirects. The build should instead use route groups so file organization can be semantic while URL paths remain 1:1:

- Blog/content implementation: `app/(migrated-content)/[slug]/page.tsx` serving old root paths like `/testosterone-dosage-women-weekly/`.
- Service-detail implementation: same root slug route, with service slugs selected from data and rendered with the service-detail template.
- Internal data grouping may still call these records `blog` and `services`, but canonical paths should not gain `/blog/` or `/services/` prefixes.

If the orchestrator forces `/blog/[slug]` or `/services/[slug]`, every moved status-200 URL becomes `REQUIRES_REDIRECT`; this is not recommended for tier1/tier2 URLs.

### Dynamic Params

Blog/content route:

- File target: `build/app/(migrated-content)/[slug]/page.tsx`.
- `generateStaticParams`: all status-200 paths from `url_inventory.json` not handled by explicit static routes.
- Canonical examples: `/5-tips-for-achieving-orgasm-through-intercourse/`, `/3d-model-of-the-clitoris-exposes-pleasurable-female-anatomy/`, `/testosterone-dosage-women-weekly/`.
- Do not add `/blog/` prefix unless the orchestrator approves redirects for all moved paths.

Service-detail subset:

- Use the same root `[slug]` URL mechanism with a service dataset flag.
- Initial service slugs to render with service-detail template, based on redirect destinations and high-intent treatment pages: `hormone-therapy`, `menopause-perimenopause`, `body-contouring-women-beverly-hills`, `vaginal-rejuvenation-expert`, `sexual-urinary-tract-health`, `emsella-treatment-for-incontinence`, `beautifill`, `weight-loss-program`, `anti-aging-treatment`, `pt-141`, `ghk-cu-glow-peptide-beverly-hills`.

6. `blog_post_migrator`: convert migrated blog/content HTML into MDX/data records. Use `url_inventory.json` for canonical path, title, description, priority, and GSC metadata. Resolve the 375 vs 376 count discrepancy first.
7. `motion_polish`: one batch using `motion_spec.json` IDs. Must query Motion MCP before writing animation code. Use `motion/react`, `useReducedMotion()`, and `LazyMotion + domAnimation` where appropriate.
8. `redirect_implementer`: write `next.config.js` redirects for the 77 existing 301s and any approved route deviations. Flatten redirect chains only if the orchestrator approves changing exact destination behavior.
9. `schema_implementer`: inject JSON-LD by route bucket and page data. Old site has zero schemas, so additions should be generated from current page facts.
10. `build_qa`: run lint, typecheck, `npm run build`, sitemap/robots validation, redirect validation with local `next dev` plus `curl -I`, and Lighthouse smoke checks.
